# Hotfix: Customer Code Auto Number

## เป้าหมาย

ปรับรหัสลูกค้าให้ระบบสร้างอัตโนมัติ ผู้ใช้ไม่ต้องกรอกเอง โดยเริ่มจาก `0001` และเพิ่มทีละ 1 เช่น `0002`, `0003`.

## สิ่งที่แก้

- Backend สร้างรหัสลูกค้าอัตโนมัติใน transaction ตอนเพิ่มลูกค้า
- ใช้ PostgreSQL advisory transaction lock เพื่อป้องกันเลขซ้ำเมื่อมีผู้ใช้เพิ่มลูกค้าพร้อมกัน
- หน้าเพิ่มลูกค้าเปลี่ยนช่องรหัสลูกค้าเป็น read-only และแสดงว่า “สร้างอัตโนมัติ”
- Frontend ไม่ส่ง `code` ไปตอนสร้าง/แก้ไขลูกค้า
- ตอนแก้ไขลูกค้า backend จะรักษารหัสเดิมไว้ ไม่ให้แก้ผ่าน API โดยตรง
- เพิ่ม migration `008_customer_code_autonumber.sql` เพื่อเติมรหัสให้ลูกค้าเดิมที่ยังไม่มีรหัส

## หมายเหตุ

ยังคงใช้คอลัมน์ `customers.code` และ unique index เดิม เพื่อไม่กระทบเอกสารเดิมและ customer snapshot.

## วิธีทดสอบ

1. รัน migration
2. เพิ่มลูกค้ารายแรก ตรวจว่าได้รหัส `0001` หากยังไม่มีลูกค้าเดิม
3. เพิ่มลูกค้ารายต่อไป ตรวจว่าได้ `0002`
4. แก้ไขข้อมูลลูกค้าเดิม ตรวจว่ารหัสไม่เปลี่ยน
5. ค้นหาลูกค้าด้วยรหัส เช่น `0001`
